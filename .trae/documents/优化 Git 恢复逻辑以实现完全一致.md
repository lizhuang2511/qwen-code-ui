**原因分析：**
1.  **忽略文件未删除**：`git clean -fd` 默认**不会删除**在 `.gitignore` 中列出的文件（如编译生成的文件、临时配置等）。如果您新建的文件恰好被规则忽略，它们就会残留。
2.  **嵌套仓库**：如果新建的是一个包含 `.git` 的子文件夹（嵌套仓库），Git 默认需要 `-ff` (双重强制) 才会删除。
3.  **完全一致性**：要达到“文件数量和文件都和那个版本完全一致”，必须清除所有 **未追踪（Untracked）** 和 **被忽略（Ignored）** 的文件。

**修改计划：**
1.  **修改 `crates/backend/git_utils.py`**：
    *   将清理命令升级为 `repo.git.clean("-ffdx")`。
        *   `-ff`: 强制删除，包括嵌套的 git 仓库。
        *   `-d`: 删除未追踪的目录。
        *   `-x`: **关键**，删除被忽略的文件（ignored files）。这能保证工作区与 Git 版本库**完全一致**，不留任何多余文件。
    *   保留 `repo.git.reset("--hard", ...)` 以重置已追踪文件的修改。

**注意**：加上 `-x` 会删除所有未提交的构建产物（如 `dist/`）、依赖库（如 `node_modules/`，如果它们被忽略且未提交）和本地配置文件（如 `.env`，如果被忽略）。这是实现“完全一致”的代价。如果您的项目中这些文件需要保留，请在确认时告知，否则我将严格执行“完全一致”的要求。
