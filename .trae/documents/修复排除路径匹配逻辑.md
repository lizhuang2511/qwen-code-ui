# 修复排除路径匹配逻辑

## 问题分析
根据用户提供的日志信息：
1.  **配置读取正确**：`[调试] 读取到的排除列表: ['D:\\qwencode\\临时计算\\文档']`。
2.  **文件未被排除**：`[调试] 添加文件: 文档\测试.md`。
3.  **核心原因**：
    *   **路径格式不匹配**：排除列表中的路径是绝对路径 `D:\qwencode\临时计算\文档`，而 `is_path_excluded` 接收的是相对路径 `文档\测试.md`。
    *   **当前逻辑缺陷**：`is_path_excluded` 函数目前的实现假设排除列表中的项也是相对路径（如 `文档` 或 `./文档`）。当排除项是绝对路径时，简单的 `normalize_path` 无法将其与相对路径 `file_rel_path` 匹配上。
    *   此外，`file_rel_path` 在日志中显示为 `文档\测试.md`（Windows 风格），而 `normalize_path` 会将其转为 `文档/测试.md`。排除项 `D:\qwencode\临时计算\文档` 标准化后变为 `D:/qwencode/临时计算/文档`。两者显然不匹配。

## 修复计划
我们需要修改 `crates/backend/version_utils.py` 和 `crates/backend/git_utils.py` 中的 `is_path_excluded` 函数逻辑。

### 1. 增强 `is_path_excluded` 逻辑
修改后的逻辑应包含以下步骤：
1.  **标准化输入路径**：`rel_path` 已经是相对路径，只需标准化分隔符。
2.  **处理排除项**：遍历 `excluded_paths` 时，需判断每一项是**绝对路径**还是**相对路径**。
    *   **如果是绝对路径**：
        *   这通常是不推荐的配置方式，因为项目移动后会失效。但既然用户配置生成了绝对路径（可能是前端传来的），我们必须处理。
        *   **策略**：尝试将绝对路径转换为相对于项目根目录的相对路径。
        *   但是，`is_path_excluded` 函数目前只接收 `rel_path` 和 `excluded_paths`，**没有接收项目根目录路径**。这导致无法正确将绝对路径转为相对路径进行比较。

### 2. 方案调整：传递项目根目录或预处理排除列表
由于 `is_path_excluded` 被多次调用，且不知道根目录，我们有两种选择：
*   **方案 A（推荐）**：在调用 `is_path_excluded` 之前（即在 `_zip_workspace` 和 `restore_version` 内部），预处理 `excluded` 列表。将所有绝对路径转换为相对于 `source_dir` 的相对路径。
*   **方案 B**：修改 `is_path_excluded` 签名，传入根目录。

鉴于 `_zip_workspace` 和 `restore_version` 都已经知道 `source_dir`（即 `path`），**方案 A** 更加高效且改动最小。

### 3. 具体实施步骤
1.  **修改 `_zip_workspace` 和 `restore_version`**：
    *   在获取到 `excluded = get_excluded_paths(source_dir)` 后，立即进行预处理。
    *   遍历 `excluded`，如果某项是绝对路径且是 `source_dir` 的子路径，则将其转换为相对路径。
    *   如果是绝对路径但不在 `source_dir` 下，则忽略或保留（反正匹配不上）。
2.  **优化 `is_path_excluded`**：
    *   保留现有的标准化和前缀匹配逻辑，因为预处理后所有排除项都将是相对路径。
    *   确保能匹配 `文档` 和 `文档/测试.md`。

## 代码修改示例
在 `crates/backend/version_utils.py` 和 `git_utils.py` 中：

```python
def _preprocess_excluded_paths(root_path: str, excluded_paths: List[str]) -> List[str]:
    """
    Convert absolute paths in excluded list to relative paths based on root_path.
    Also normalize all paths.
    """
    processed = []
    root_abs = os.path.abspath(root_path)
    
    for path in excluded_paths:
        if not path:
            continue
            
        # Check if absolute
        if os.path.isabs(path):
            # If it's inside root, make it relative
            try:
                # commonpath check to ensure it's actually inside
                if os.path.commonpath([root_abs, os.path.abspath(path)]) == root_abs:
                    rel = os.path.relpath(path, root_abs)
                    processed.append(normalize_path(rel))
                else:
                    # Outside root, ignore or keep (won't match internal files anyway)
                    pass 
            except ValueError:
                pass
        else:
            processed.append(normalize_path(path))
            
    return processed

# ... 在 _zip_workspace 和 restore_version 中调用此函数 ...
excluded = get_excluded_paths(source_dir)
excluded = _preprocess_excluded_paths(source_dir, excluded) # 新增
# ... 之后继续使用 is_path_excluded ...
```

这样，`D:\qwencode\临时计算\文档` 会被转换为 `文档`，然后 `is_path_excluded` 就能正确匹配 `文档/测试.md` 了。
