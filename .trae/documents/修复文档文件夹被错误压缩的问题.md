# 修复文档文件夹被错误压缩的问题

## 问题分析
用户反馈“文档”文件夹被错误压缩，尽管已在 `config.json` 中配置排除。
经过检查代码和用户描述，可能存在以下原因：
1.  **路径标准化问题**：`is_path_excluded` 函数中虽然进行了标准化，但对于中文路径的处理可能在不同操作系统（特别是 Windows）下存在编码或路径分隔符的微妙差异。
2.  **根目录处理**：在 `_zip_workspace` 中，`rel_root` 如果是 `.` 被置为空字符串。当遍历顶层目录时，`d` 就是目录名。如果 `excluded` 列表中包含 `"文档"`，理论上 `is_path_excluded` 应该返回 `True`。但如果 `excluded` 列表中的路径格式与 `d` 不一致（例如包含 `./` 前缀），则可能匹配失败。
3.  **配置读取确认**：虽然代码已修改为从根目录读取 `config.json`，但为了双重保险，我们需要确保在所有相关操作中都使用了正确的配置路径，并且正确处理了 JSON 解码错误（尽管目前已处理）。

## 修复计划

### 1. 增强路径排除匹配逻辑
-   修改 `is_path_excluded` 函数，使其更加健壮：
    -   支持匹配带有 `./` 前缀的排除项（用户可能手写了 `./文档`）。
    -   支持匹配尾部带有 `/` 或 `\` 的排除项。
    -   确保在比较前，排除项和目标路径都经过彻底的标准化（去除首尾空白、统一分隔符、去除开头的 `./`）。

### 2. 优化压缩遍历逻辑
-   在 `_zip_workspace` 中，确保传入 `is_path_excluded` 的路径是干净的相对路径。
-   增加调试日志（虽然用户看不到，但在开发测试中有用），或者在逻辑中增加更显式的检查。

### 3. 编写针对性测试脚本
-   创建一个包含中文文件夹“文档”的测试环境。
-   在 `config.json` 中配置排除“文档”。
-   执行备份操作，验证 Zip 包中是否包含“文档”文件夹。
-   此脚本将用于验证修复是否生效。

### 4. 代码同步
-   确保 `crates/backend/version_utils.py` 和 `crates/backend/git_utils.py` 保持一致。

## 具体修改点
在 `is_path_excluded` 中：
```python
def is_path_excluded(rel_path: str, excluded_paths: List[str]) -> bool:
    if not excluded_paths:
        return False
        
    # Normalize input path: unify separators, strip whitespace
    norm_path = rel_path.replace("\\", "/").strip().rstrip("/")
    # Remove leading ./ if present
    if norm_path.startswith("./"):
        norm_path = norm_path[2:]
    
    for ex in excluded_paths:
        # Normalize exclusion pattern
        ex_norm = ex.replace("\\", "/").strip().rstrip("/")
        if ex_norm.startswith("./"):
            ex_norm = ex_norm[2:]
            
        if not ex_norm:
            continue
            
        # Exact match
        if norm_path == ex_norm:
            return True
            
        # Directory prefix match
        if norm_path.startswith(ex_norm + "/"):
            return True
            
    return False
```
这将大大提高匹配的容错性，解决因配置写法不同导致的排除失效问题。
