我将修复文件删除时的权限错误 `[WinError 5] 拒绝访问`。

**原因分析：**
错误发生在 `restore` 函数清理工作区时。`os.remove` 或 `shutil.rmtree` 尝试删除 `.git` 目录下的只读文件（Git 对象文件通常是只读的）时失败。虽然我们已经移除了 Git 管理，但工作区中可能仍残留 `.git` 文件夹（尤其是之前的 Git 仓库遗留），或者有其他只读文件。

**修复方案：**
1.  **增强删除逻辑**：在 `crates/backend/git_utils.py` 中，修改文件删除逻辑。
2.  **处理只读文件**：为 `shutil.rmtree` 添加 `onexc` (或 `onerror`，取决于 Python 版本，Python 3.12+ 推荐 `onexc`，旧版用 `onerror`) 回调，在遇到“拒绝访问”错误时，尝试修改文件权限为可写（`stat.S_IWRITE`），然后重试删除。
3.  **忽略 `.history`**：确保删除逻辑依然严格跳过 `.history` 目录。

**具体实现**：
在 `git_utils.py` 中定义一个 `remove_readonly` 辅助函数，并将其传递给 `shutil.rmtree`。对于单个文件删除 `os.remove`，如果失败也尝试修改权限后重试。
