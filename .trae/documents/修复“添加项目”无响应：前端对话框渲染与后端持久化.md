## 变更目标
- 移除所有 Tauri 依赖，前端与后端统一走 Pywebview 桥接。
- “添加项目”按钮直接调用 Pywebview 原生文件夹选择对话框，选中后完成新增与列表刷新。

## 前端改造
- 删除 Tauri 相关逻辑与依赖：
  - 去除 `@tauri-apps/plugin-dialog` 动态导入与使用（`frontend/src/pages/Projects.tsx:111-116`）。
  - 去除 `window.__TAURI__` 环境判断（`frontend/src/pages/Projects.tsx:105-110`）。
  - 去除仅 Web 模式下渲染的目录选择对话框门槛 `{__WEB__ && <DirectorySelectionDialog .../>}`（`frontend/src/pages/Projects.tsx:262-269`）。
- 改为调用 Pywebview 原生文件对话框：
  - 在 `frontend/src/lib/api.ts` 的 `API` 接口中新增 `select_directory(): Promise<string | null>`。
  - 在 `frontend/src/pages/Projects.tsx` 新增 `handleAddProjectPy`：
    - 调用 `const selectedPath = await api.select_directory()`；
    - 若有返回则调用现有 `handleAddProject(selectedPath)` 完成 `get_canonical_path → get_project → refresh → navigate` 链路（`frontend/src/pages/Projects.tsx:68-98`）。
  - 将按钮 `onClick` 改为直接调用 `handleAddProjectPy`（替换 `__WEB__ ? setDirectoryDialogOpen(true) : handleAddProjectNative`，位置：`frontend/src/pages/Projects.tsx:152-156`）。
- 保留并使用已有的 Pywebview 桥接：`frontend/src/lib/api.ts:171-195` 与 `frontend/src/lib/pywebviewApi.ts:8-19`。

## 后端改造
- 在 `crates/backend/api.py` 新增：
  - `def select_directory(self) -> Optional[str]`：
    - 使用 `import webview` 并调用 `webview.create_file_dialog(webview.FOLDER_DIALOG, allow_multiple=False)` 或 `webview.windows[0].create_file_dialog(...)`，返回单个目录路径或 `None`。
  - 遵守“禁止使用 try/except”，使用显式条件分支。
- 持久化新增项目：
  - 在 `crates/projects.py` 新增 `upsert_project(project_id: str, path: str, title: str) -> None`：
    - 确保 `data/projects.json` 文件存在（沿用 `_ensure_data_dir`），存在则更新，否则追加；不使用 `try/except`。
  - 在 `crates/backend/api.py:get_project (95-104)` 中调用 `projects.upsert_project(...)` 写入后返回与 `list_enriched_projects` 一致的结构（`crates/backend/api.py:92-93`）。

## 验证
- 操作验证：
  - 点击“添加项目”→ 弹出系统原生文件夹选择对话框（Pywebview）；选择目录后项目列表刷新并跳转详情（`frontend/src/pages/Projects.tsx:88-92`）。
- 数据验证：
  - `data/projects.json` 出现包含新 `id` 与 `path` 的条目；`list_enriched_projects` 可读取到该条目（`crates/projects.py:22-36`）。
- 环境验证：
  - 前端所有 API 均通过 `window.pywebview.api.*` 调用后端；不再访问 Tauri 或 Web REST。

## 约束
- 全部 Python 改动不引入 `try/except`；前端删除 Tauri 依赖，保持纯 Pywebview 工作流。
