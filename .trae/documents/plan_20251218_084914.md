The frontend logs show two critical issues:

1.  **"Tool call not found for update: null"** (repeated many times): This means the frontend is receiving `tool_call_update` events, but it cannot find a corresponding tool call in its internal state to update. This leads to the UI showing a tool call as perpetually "running".
2.  **"Dialog is changing from uncontrolled to controlled"**: This is a React state management issue, likely caused by the tool call update logic trying to modify state that doesn't exist.

**Root Cause:**
The backend (`session.py`) is correctly receiving and parsing `tool_call_update` events and forwarding them via `acp-session-update-{sessionId}`. However, the frontend's `updateConversation` function (in `useConversationEvents.ts`) is trying to update a tool call with `toolCallId`, but this ID does not exist in the current conversation's messages. This happens if:
*   The initial `tool_call` event (which creates the tool call in the UI state) was missed or not processed correctly.
*   The `toolCallId` in the `tool_call_update` event is `null` or mismatched.

**Plan:**

1.  **Add Defensive Checks in Frontend (`useConversationEvents.ts`)**:
    *   In the `tool_call_update` handler (around line 525), add a check: if `update.toolCallId` is `null` or empty, log a warning and **return early** instead of trying to update a non-existent tool call. This will prevent the flood of errors.
    *   Add a check: if the `toolCallId` is valid but the tool call is *still* not found after the loop, log a detailed error including the `toolCallId` and the list of existing tool call IDs in the conversation. This will help diagnose if the IDs are mismatched.

2.  **Add Logging in Backend (`session.py`)**:
    *   In the `tool_call_update` handler, log the full `toolCallId` and `status` being sent to the frontend. This will help correlate backend events with frontend errors.

This will stop the error spam and provide the necessary information to fix the underlying state mismatch, which is the true cause of the "stuck in running" UI.
