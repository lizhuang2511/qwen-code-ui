# 修复同名文件/文件夹导致无法创建文件的问题

## 问题澄清与分析 (Clarification & Analysis)
您提到“文件不是 qwencli 在操作吗？”，这是一个很好的切入点。
- **qwencli 的角色**：确实，`qwencli` 负责执行 Agent 的命令（如终端中运行的 `python gear_calculator_example.py`）。它是一个独立的进程，拥有自己的文件写入逻辑。
- **IDE/后端 (filesystem.py) 的角色**：`crates/filesystem.py` 主要负责 **IDE 界面**（如文件资源管理器）发起的文件操作（新建、保存等）。
- **问题定位**：您遇到的“新建文件无法创建”问题，如果是指通过 IDE 界面操作（例如右键新建文件），那么问题确实出在 `filesystem.py`。因为该文件中的 `write_file_content` 函数直接尝试写入，未处理“目标路径已存在且为文件夹”的情况，导致操作失败。

即使 `qwencli` 在某些情况下（如工具调用）也可能写入文件，但我们能控制和修复的是 Python 后端代码。修复 `filesystem.py` 可以解决 IDE 界面操作时的报错问题，提升用户体验。

## 修复计划 (Repair Plan)
我们将修改 `crates/filesystem.py`，在写入文件前增加对目标路径类型的检查。

### 预期代码变更
在 `crates/filesystem.py` 中修改 `write_file_content` 函数：

```python
def write_file_content(path: str, content: str) -> Dict:
    p = Path(path)
    
    # 修复：检查是否为文件夹，防止写入失败
    # 遵循用户规则：不使用 try-except
    if p.exists() and p.is_dir():
        return {
            "path": str(p),
            "content": None,
            "size": 0,
            "modified": 0,
            "encoding": "utf-8",
            "is_text": False,
            "is_binary": False,
            "error": "Target path is a directory"
        }
        
    p.write_text(content, encoding="utf-8")
    return read_file_content(str(p))
```

此修复仅影响 IDE/后端的文件操作逻辑，不会干扰 `qwencli` 的内部行为，是安全且必要的修正。
