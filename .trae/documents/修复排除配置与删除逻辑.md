# 修复排除配置与删除逻辑（含压缩备份优化）

## 问题分析
1.  **排除规则失效**：在恢复版本（`restore`）时，计算文件相对路径时未正确处理根目录（`.`），导致生成的路径与排除列表不匹配，从而误删了本应排除的文件。
2.  **压缩备份排除不严谨**：用户指出压缩备份时也需要严格排除。虽然现有代码有部分逻辑，但可能存在路径处理不一致（如未处理 `.` 或路径分隔符）的问题，导致某些应排除的文件仍被打包。
3.  **删除逻辑缺陷**：原有的删除逻辑依赖 `shutil.rmtree` 和 `try-except`，无法精细地保留目录下的某些被排除文件。
4.  **代码规范**：代码中大量使用 `try-except`，需替换为防御性编程。

## 修复计划
我们将重点修改 `crates/backend/version_utils.py` 文件（若 `git_utils.py` 被使用，也将同步修复）。

### 1. 统一路径计算与排除检查
-   **标准化路径处理**：创建一个辅助函数来处理相对路径计算，确保根目录始终被视为空字符串（而非 `.`），并统一使用正斜杠 `/` 作为分隔符。
-   **增强 `is_path_excluded`**：确保该函数能准确匹配精确路径和子目录路径，无论操作系统如何。

### 2. 优化压缩备份逻辑（`_zip_workspace`）
-   审查并修正 `_zip_workspace` 函数。
-   应用标准化的路径处理逻辑，确保在遍历目录时，被排除的文件夹和文件**绝对不会**被写入 Zip 包。
-   确保在遍历目录结构时（`os.walk`），一旦遇到被排除的目录，立即停止进入该目录（剪枝），提高效率并防止误包含。

### 3. 重构恢复与删除逻辑（`restore_version`）
采用精细化的三步策略：
1.  **第一步：选择性删除**
    -   使用 `os.walk(topdown=True)` 遍历工作区。
    -   **剪枝**：遇到被排除的目录，直接从遍历列表中移除，**保留**该目录及其内容。
    -   **文件处理**：仅删除未被排除的文件；被排除的文件保持原样。
2.  **第二步：清理空目录**
    -   再次遍历，删除那些因文件被删而变空的目录（前提是该目录本身未被排除）。
3.  **第三步：选择性解压**
    -   遍历 Zip 包，解压前检查每个文件是否被排除。如果被排除（即使 Zip 里有），也不覆盖本地文件。

### 4. 移除 `try-except` 语句
-   全面重构代码，移除所有用于流程控制的 `try-except`。
-   使用 `os.path.exists`、`os.listdir` 等进行前置检查。
-   对于文件读写，确保在操作前验证环境，而非捕获错误。

### 5. 确认配置路径
-   确保 `_get_config_path` 指向项目根目录下的 `config.json`，确保配置读取正确。

通过以上优化，确保**备份**和**恢复**两个环节都严格遵守排除规则，且代码逻辑清晰、健壮。
