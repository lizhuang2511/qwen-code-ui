## 概览
- 两种运行形态：Web 模式（浏览器 + 后端 HTTP/WS 服务）与桌面模式（Tauri 桌面应用）。
- 业务逻辑不重复：二者共享同一个 `backend` crate 的核心实现，通过不同事件发射器与传输层对外暴露。
- 结论：运行时可独立部署与使用，但逻辑层并非相互独立，特性由同一后端实现镜像提供。

## 架构共享
- 共用核心：`GeminiBackend<E>` 在两种模式下复用，仅事件发射器类型不同。
- 桌面事件发射器：`crates/tauri-app/src/event_emitter.rs:17-23` 的 `TauriEventEmitter` 实现 `EventEmitter`。
- Web 事件发射器：`crates/server/src/main.rs:140-197` 的 `WebSocketsEventEmitter` 基于有序队列广播。
- 会话与 CLI 交互：统一走标准 IO 的 JSON-RPC（ACP）。初始化与消息处理在 `crates/backend/src/session/mod.rs:368-448, 940-1144`。

## 运行时独立性
- Web 模式独立运行：
  - 启动 Rocket 服务，挂载 REST 与 WebSocket 路由，前端静态资源内嵌并直接服务。
  - 关键入口：`crates/server/src/main.rs:877-932`（`rocket()`）与 `834-875`（`/api/ws`）。
- 桌面模式独立运行：
  - 启动 Tauri 应用，前端通过 `invoke` 调用命令，事件通过 Tauri 事件通道下发。
  - 命令注册：`crates/tauri-app/src/commands/mod.rs:14-606`；事件发射：`crates/tauri-app/src/event_emitter.rs:17-23`。
- 互不依赖：任意一方可在没有另一方的情况下提供完整交互。但两者都依赖同一后端库与同类 CLI 通道。

## 事件与通讯
- 后端→前端事件名统一，按模式选用不同发射器：
  - 示例事件：`cli-io-{session_id}`、`ai-output-{session_id}`、`acp-session-update-{session_id}`、`process-status-changed`。
  - 事件枚举与载荷：`crates/backend/src/events/mod.rs:8-60, 62-111`。
- Web 模式传输：
  - REST 路由镜像后端能力，如 `POST /api/start-session`、`/api/send-message` 等，见 `crates/server/src/main.rs:382-828`。
  - WebSocket 推送事件：`/api/ws`，见 `crates/server/src/main.rs:834-875`。
- 桌面模式传输：
  - Tauri 命令镜像同等能力：`start_session`、`send_message`、`get_process_statuses` 等，见 `crates/tauri-app/src/commands/mod.rs:23-172, 523-607`。

## 前端适配
- Web 前端：通过 `axios` 调用 `/api/*`，通过浏览器 `WebSocket` 订阅事件；与桌面事件名保持一致，便于代码复用。
- 桌面前端：通过 Tauri `invoke` 直接调用命令，通过 `app_handle.emit` 订阅事件，事件结构与 Web 模式一致。
- 结果：前端业务组件可在少量适配层差异下复用。

## 部署与打包
- Web 模式：单独部署 Rocket 服务即可对外提供 HTTP/WS 与静态前端。
- 桌面模式：Tauri 打包本机应用，运行时不依赖外部 Web 服务；可选包含 sidecar 服务用于特定场景（配置在 `tauri.conf.json`）。
- 共享依赖：两者都运行同类外部 CLI（Gemini/Qwen/LLxprt），通过标准 IO JSON-RPC（ACP）交互，见 `crates/backend/src/rpc/mod.rs:28-52, 161-173` 与 `crates/backend/src/session/mod.rs:368-448`。

## 结论
- 独立性：两种模式在进程与部署层面完全独立，可分别启动、升级与使用。
- 非重复实现：核心后端逻辑统一，API 与事件在两种模式下镜像暴露，避免功能分叉。
- 迁移与维护：新增后端能力只需在共享 `backend` crate 实现，然后在两端各自映射（REST 路由或 Tauri 命令）。
