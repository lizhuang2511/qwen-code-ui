# Python Wrapper 与 Qwen CLI 通讯方式说明

## 概述

Python Wrapper 是一个用于与 Qwen CLI 进行交互的中间层，通过封装命令行调用的方式实现 Python 代码与 Qwen 模型的通讯。本说明文档详细介绍了通讯机制、实现原理和使用方法。

## 通讯架构

```
Python 应用程序
        ↓
Python Wrapper (qwen_python_wrapper.py)
        ↓
系统命令行调用 (subprocess)
        ↓
Qwen CLI (命令行接口)
        ↓
Qwen 模型服务
```

## 核心通讯机制

### 1. 基于 Subprocess 的命令调用

Python Wrapper 使用 Python 的 `subprocess` 模块来执行 Qwen CLI 命令，实现与底层服务的通讯。

#### 基本调用流程
```python
def _run_command(self, args: List[str], input_text: Optional[str] = None, timeout: int = 300):
    cmd = [self.cli_path] + args
    result = subprocess.run(
        cmd,
        input=input_text,
        text=True,
        capture_output=True,
        timeout=timeout,
        check=False
    )
    return result
```

#### 参数传递机制
- **命令参数**: 通过 `args` 列表传递 CLI 参数
- **输入内容**: 通过 `input_text` 参数传递输入文本
- **超时控制**: 默认 300 秒超时保护
- **错误处理**: 手动处理返回码和错误信息

### 2. CLI 路径发现和兼容性

Wrapper 实现了智能的 CLI 路径发现机制，支持多种安装方式：

#### 路径搜索策略
1. **直接路径**: 用户指定的 CLI 路径
2. **系统 PATH**: 使用 `shutil.which()` 查找
3. **PowerShell 检测**: Windows 系统下通过 PowerShell 获取命令路径
4. **常见安装位置**: 预定义的常见安装路径列表

#### 支持的平台
- **Windows**: `.cmd`, `.ps1`, `.exe` 格式
- **Unix/Linux/macOS**: 可执行文件格式
- **Node.js 全局安装**: npm 全局包路径

### 3. 认证机制

#### 凭证管理
- **凭证文件位置**: `~/.qwen/oauth_creds.json`
- **凭证验证**: 检查文件存在性和 JSON 格式有效性
- **自动检测**: 初始化时自动验证凭证状态

#### 认证流程
```python
def check_credentials(self) -> bool:
    if not os.path.exists(self.credentials_path):
        return False
    
    # 验证 JSON 格式
    try:
        with open(self.credentials_path, 'r', encoding='utf-8') as f:
            content = f.read().strip()
            if not content:
                return False
            json.loads(content)
        return True
    except (json.JSONDecodeError, IOError):
        return False
```

```python

```

#### JSON 响应解析
```python
result = self._run_command(args)
try:
    return json.loads(result.stdout)
except json.JSONDecodeError as e:
    raise QwenCLIError(f"Failed to parse JSON response: {e}")
```

#### 流式响应处理
```python

    行 = 行.strip()
    if 行:
```

### 3. 错误处理机制

#### 错误类型分类
1. **CLI 未找到错误**: `FileNotFoundError`
2. **命令执行错误**: 非零返回码
3. **超时错误**: `subprocess.TimeoutExpired`
4. **JSON 解析错误**: `json.JSONDecodeError`

#### 错误信息提取
```python
if result.returncode != 0:
    error_msg = result.stderr if result.stderr else result.stdout
    raise QwenCLIError(f"Qwen CLI error (exit code {result.returncode}): {error_msg}")
```

## 高级功能实现

### 1. 流式输出支持

#### 实时输出机制
```python
def 流式查询(self, prompt: str, 模型: Optional[str] = None):
    args = ["--prompt", prompt, "--stream", "true"]
    if 模型:
        args.extend(["--model", 模型])
    
    process = subprocess.Popen(
        [self.cli_path] + args,
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE,
        text=True,
        bufsize=1,
        universal_newlines=True
    )
    
    完整响应 = ""
    for 行 in process.stdout:
        print(行, flush=True)  # 实时显示
        完整响应 += 行 + "\n"
    
    return 完整响应.strip()
```

#### 兼容性处理
- **流式支持检测**: 自动检测 CLI 是否支持流式输出
- **回退机制**: 不支持时自动回退到普通查询模式
- **编码处理**: 支持 Unicode 和中文字符

## 故障排除

### 常见问题

#### 1. CLI 未找到
**错误信息**: `Qwen CLI not found at path: qwen`

**解决方案**:
1. 安装 Qwen CLI: `npm install -g @qwen/cli`
2. 检查 PATH 配置
3. 指定完整路径创建 Wrapper

#### 2. 认证失败
**错误信息**: `Qwen CLI credentials not found`

**解决方案**:
1. 运行认证: `qwen login`
2. 检查凭证文件: `~/.qwen/oauth_creds.json`
3. 验证 JSON 格式有效性

#### 3. 超时错误
**错误信息**: `Qwen CLI command timed out after 300 seconds`

**解决方案**:
1. 增加超时时间参数
2. 检查网络连接
3. 简化查询内容
